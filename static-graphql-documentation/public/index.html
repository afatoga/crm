Query
query PartyRelationships(
  $appUserGroupId: Int!,
  $partyId: Int!
) {
  partyRelationships(
    appUserGroupId: $appUserGroupId,
    partyId: $partyId
  ) {
    organizationToOrganization {
      id
      typeId
      firstPartyId
      firstPartyTypeId
      secondPartyId
      secondPartyTypeId
      firstPartyName
      secondPartyName
    }
    personToOrganization {
      id
      typeId
      firstPartyId
      firstPartyTypeId
      secondPartyId
      secondPartyTypeId
      firstPartyName
      secondPartyName
    }
    personToPerson {
      id
      typeId
      firstPartyId
      firstPartyTypeId
      secondPartyId
      secondPartyTypeId
      firstPartyName
      secondPartyName
    }
  }
}

      "organizationToOrganization": [
        ExtendedPartyRelationship
      ],

  }

ID

Description

The ID scalar type represents a unique identifier, often used to refetch an object or as key for a cache. The ID type appears in a JSON response as a String; however, it is not intended to be human-readable. When expected as an input type, any string (such as "4") or integer (such as 4) input value will be accepted as an ID.
